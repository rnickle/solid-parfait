Consideration

After some reflection, I do not think I’ll be doing any work soon that needs to run natively in Windows. I am more likely to target Linux and specifically Ubuntu. Therefore, WSL with VS Code is the natural ecosystem for my current work.

Notes on configuring a WSL environment.

  • build-essential
    • We will need this prerequisite and more for various parts of our toolchain. The recommended list for pyenv is fairly comprehensive, let’s get that installed: 
      $ sudo apt-get install -y make build-essential libssl-dev zlib1g-dev libbz2-dev libreadline-dev libsqlite3-dev wget curl libncurses5-dev libncursesw5-dev xz-utils tk-dev libffi-dev liblzma-dev
  • git
    • Many prefer the use of main to indicate the default branch. The version of git that ships with the current WSL default Ubuntu does not follow that practice. Easy to fix before creating any additional repositories: 
      $ git config --global init.defaultBranch main
  • golang
    • Avoiding Aptitude in favor of a local install: 
      $ cd /usr/local
$ VER=1.18.1
$ wget https://dl.google.com/go/go${VER}.linux-amd64.tar.gz
$ sudo tar -C /usr/local -xzf go${VER}.linux-amd64.tar.gz
$ rm go${VER}.linux-amd64.tar.gz
    • In order to get binaries built with go on the path, it is necessary to set and export GOPATH and add the bin directory under GOPATH to your PATH. This is best done for VS Code users in $HOME/.vscode-server/server-env-setup: 
      #!/bin/bash

if [ -d "$HOME/local/go" ]; then
  export GOPATH="/usr/local/go"
  PATH="${PATH}:${GOPATH}/bin"
fi
      It is necessary to restart VS Code to have it re-read server-env-setup.
    • To make it easier for the utility user in WSL to write to the GOPATH: 
      $ sudo mkdir /usr/local/go
$ sudo chown -R utility: /usr/local/go
  • rust
    • Again, avoiding Aptitude, install as user mode rustup: 
      $ curl https://sh.rustup.rs -sSf | sh
      And make sure we add it to path: 
      if [ -d "$HOME/.cargo/bin" ]; then
  PATH="${PATH}:${HOME}/.cargo/bin"
fi
  • python
    • We will almost definitely be testing against multiple versions of Python, so instead of installing with Aptitude package installer, we will use the excellent pyenv. The requirements listed above in build-essential are necessary for this: 
      $ curl -L https://github.com/pyenv/pyenv-installer/raw/master/bin/pyenv-installer | bash
    • Adopt the instructions for setting up .profile for the VS Code environment, add to $HOME/.vscode-server/server-env-setup: 
      export PYENV_ROOT="$HOME/.pyenv"
command -v pyenv >/dev/null || export PATH="$PYENV_ROOT/bin:$PATH"
eval "$(pyenv init -)"
    • After restarting the VS Code environment, install the Python interpreters: This takes a little time since we’re building from source: 
      $ pyenv install 3.9 3.10 3.11 3.12
    • And set our default global Pythons: 
      $ pyenv global 3.11 3.12.0a3 3.10 3.9
$ python --version
Python 3.11.1
