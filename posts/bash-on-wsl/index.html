Bash on WSL in vs Code

When customizing the Bash environment in a “remote” session with WSL, it is important to remember that VS Code Remote does not treat the session as a login session.

Instead, add a server-env-setup file in $HOME/.vscode-server/. In this example I would like to use the convention of having the go environment installed locally:

$ cat $HOME/.vscode-server/server-env-setup
#!/bin/bash

if [ -d "$HOME/local/go" ]; then
    PATH="${PATH}:/usr/local/go/bin"
fi

Now the go environment will be added to the PATH when the vscode folder is opened in WSL.
